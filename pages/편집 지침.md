== 개요 ==

'''편집 지침'''은 산삼위키에서 문서를 새로 만들거나 수정할 때 따라야 할 기본 원칙과 문법을 설명하는 문서이다.

산삼위키는 게임 커뮤니티를 위한 자체 위키이며, 여러 사용자가 같은 문서를 함께 편집한다. 따라서 문서는 개인 게시글처럼 작성하기보다 다른 사용자가 읽고 이어서 수정하기 쉬운 형태로 작성해야 한다.

이 문서에서는 다음 내용을 설명한다.

 * 문서 작성의 기본 원칙
 * 문서 제목과 문단 구성
 * 글자 꾸미기와 링크
 * 목록, 인용문, 각주
 * 표, 접기, 코드, 수식
 * 파일과 영상
 * 틀과 매크로
 * 편집 요약과 문서 역사
 * 문서 이동, 삭제, ACL
 * 금지되는 편집 행위

[clearfix]

== 기본 원칙 ==

=== 정확하게 작성하기 ===

확인되지 않은 내용을 사실처럼 단정하지 않는다.

공식 공지, 게임 내 실제 정보, 개발자 발언, 신뢰할 수 있는 자료 등을 우선하여 작성한다.

확실하지 않은 정보는 다음과 같이 표현한다.

{{{
현재 정확한 수치는 확인되지 않았다.
}}}

{{{
일부 이용자 사이에서는 이러한 추측이 있으나 공식적으로 확인된 내용은 아니다.
}}}

출처가 필요한 내용에는 각주를 추가하는 것을 권장한다.

=== 중립적으로 작성하기 ===

문서는 특정 이용자, 길드, 세력, 게임, 운영진을 무조건 칭찬하거나 비난하는 용도로 작성하지 않는다.

잘못된 예시:

{{{
이 게임은 무조건 망한 쓰레기 게임이다.
}}}

올바른 예시:

{{{
출시 초기에는 서버 불안정과 최적화 문제가 지적되었다.
}}}

주관적인 평가는 객관적인 사실과 구분하여 작성한다.

=== 읽기 쉽게 작성하기 ===

긴 내용을 한 문단에 몰아넣지 않는다.

내용이 바뀌는 지점마다 문단 제목을 추가한다.

 * 한 문단에는 하나의 주제를 중심으로 작성한다.
 * 지나치게 긴 문장은 나눈다.
 * 같은 내용을 여러 번 반복하지 않는다.
 * 표와 목록은 필요한 경우에만 사용한다.
 * 지나친 색상, 굵기, 큰 글자 사용을 피한다.

=== 다른 이용자의 편집을 존중하기 ===

다른 이용자가 작성한 내용을 이유 없이 삭제하지 않는다.

내용을 크게 삭제하거나 구조를 변경할 때는 편집 요약에 이유를 적는다.

의견 충돌이 발생하면 반복해서 되돌리기보다 토론이나 운영진 판단을 이용한다.

=== 저작권을 지키기 ===

다른 사이트의 글을 그대로 복사하지 않는다.

외부 자료는 필요한 부분만 요약하고 출처를 남긴다.

이미지와 파일은 사용 가능한 라이선스인지 확인한 뒤 업로드한다.

[clearfix]

== 문서 제목 ==

문서 제목은 문서의 내용을 가장 잘 나타내는 이름으로 작성한다.

=== 제목 작성 원칙 ===

 * 공식 명칭이 있으면 공식 명칭을 우선한다.
 * 불필요한 수식어를 붙이지 않는다.
 * 검색하기 어려운 약칭만 사용하지 않는다.
 * 같은 대상을 설명하는 문서가 이미 있는지 검색한다.
 * 제목 끝에 마침표를 붙이지 않는다.

잘못된 예시:

{{{
진짜 엄청 강한 최강 보스!!!
}}}

올바른 예시:

{{{
검은 산의 수호자
}}}

=== 동음이의어 ===

이름이 같은 대상이 여러 개 있으면 괄호를 붙여 구분할 수 있다.

{{{
아틀라스(게임)
아틀라스(등장인물)
아틀라스(길드)
}}}

=== 넘겨주기 ===

약칭, 옛 이름, 다른 표기에서 대표 문서로 이동하게 하려면 넘겨주기를 사용한다.

{{{
#redirect 문서명
}}}

예시:

{{{
#redirect 산삼위키
}}}

넘겨주기 문법은 문서의 첫 줄에 작성한다.

[clearfix]

== 문단 구성 ==

문단 제목은 양쪽에 같은 개수의 등호를 붙여 작성한다.

{{{
 == 큰 문단 ==
 === 하위 문단 ===
 ==== 더 작은 문단 ====
 ===== 4단계 문단 =====
 ====== 5단계 문단 ======
}}}

산삼위키에서는 문단 구조를 바탕으로 목차가 자동 생성된다.

문단 단계는 순서대로 사용하는 것이 좋다.

잘못된 예시:

{{{
 == 개요 ==
 ==== 역사 ====
}}}

올바른 예시:

{{{
 == 개요 ==
 === 역사 ===
}}}

=== 문단별 편집 ===

각 문단 오른쪽의 `[편집]` 링크를 누르면 해당 문단만 수정할 수 있다.

문단 편집 주소는 다음과 같은 형식으로 표시된다.

{{{
?section=N
}}}

`N`은 편집하려는 문단 번호이다.

[clearfix]

== 목차 ==

문서에 목차를 표시하려면 다음 매크로를 사용한다.

{{{
[목차]
}}}

목차는 일반적으로 문서 상단의 개요 문단 앞에 배치한다.

문단 구조에 따라 번호가 자동으로 붙는다.

{{{
1. 개요
2. 역사
2.1. 설립
2.2. 발전
3. 특징
}}}

문단 제목을 지나치게 길게 작성하면 목차를 읽기 어려워질 수 있다.

[clearfix]

== 글자 꾸미기 ==

=== 굵게 ===

글자 양쪽에 작은따옴표 세 개를 붙인다.

{{{
'''굵은 글자'''
}}}

결과: '''굵은 글자'''

=== 기울임 ===

글자 양쪽에 작은따옴표 두 개를 붙인다.

{{{
''기울임 글자''
}}}

결과: ''기울임 글자''

=== 밑줄 ===

글자 양쪽에 밑줄 기호 두 개를 붙인다.

{{{
__밑줄 글자__
}}}

결과: __밑줄 글자__

=== 취소선 ===

글자 양쪽에 물결표 두 개를 붙인다.

{{{
~~취소된 내용~~
}}}

결과: ~~취소된 내용~~

취소선은 농담이나 장식보다 실제로 폐기되었거나 수정된 정보를 나타내는 경우에 사용하는 것이 좋다.

=== 위 첨자 ===

{{{
^^위 첨자^^
}}}

예시:

{{{
2^^10^^
}}}

=== 아래 첨자 ===

{{{
,,아래 첨자,,
}}}

예시:

{{{
H,,2,,O
}}}

=== 여러 효과 함께 사용하기 ===

여러 문법을 겹쳐 사용할 수 있다.

{{{
'''__굵고 밑줄이 있는 글자__'''
''~~기울임 취소선~~''
}}}

지나친 중첩은 가독성을 떨어뜨리므로 필요한 경우에만 사용한다.

[clearfix]

== 줄바꿈 ==

강제로 줄을 바꾸려면 `[br]`을 사용한다.

{{{
첫 번째 줄[br]두 번째 줄
}}}

표 안이나 제목 칸 안에서 줄바꿈할 때 유용하다.

일반 본문에서는 빈 줄을 넣어 문단을 구분하는 것이 좋다.

[clearfix]

== 목록 ==

=== 순서 없는 목록 ===

항목 앞에 `*`를 붙인다.

{{{
 * 첫 번째 항목
 * 두 번째 항목
  * 하위 항목
   * 더 아래 항목
}}}

목록 단계는 앞쪽 공백으로 구분한다.

=== 순서 있는 목록 ===

숫자와 마침표를 사용한다.

{{{
 1. 첫 번째 항목
 2. 두 번째 항목
  1. 하위 항목
}}}

목록 안에 지나치게 긴 설명을 넣는 것보다 필요한 경우 별도 문단으로 나누는 것이 좋다.

[clearfix]

== 인용문 ==

문장 앞에 `>`를 붙이면 인용문이 된다.

{{{
> 이것은 인용문이다.
}}}

여러 단계로 중첩할 수 있다.

{{{
> 첫 번째 인용
>> 두 번째 인용
>>> 세 번째 인용
}}}

외부 발언을 인용할 때는 인용문 아래에 출처를 적는 것을 권장한다.

{{{
> 업데이트는 6월 중 진행될 예정이다.

— 공식 개발자 공지
}}}

긴 글 전체를 그대로 옮기기보다 필요한 부분만 인용한다.

[clearfix]

== 내부 링크 ==

다른 산삼위키 문서로 연결하려면 문서 이름을 대괄호 두 쌍으로 감싼다.

{{{
[[문서명]]
}}}

예시:

{{{
[[산삼위키]]
}}}

=== 표시 이름 변경 ===

실제 연결되는 문서명과 화면에 표시되는 글자를 다르게 할 수 있다.

{{{
[[산삼위키|이 위키]]
}}}

이 경우 `산삼위키` 문서로 연결되지만 화면에는 `이 위키`라고 표시된다.

=== 문서 내부 링크 ===

현재 문서의 특정 문단이나 앵커로 이동하려면 `#`을 사용한다.

{{{
[[#역사]]
[[#역사|역사 문단으로 이동]]
}}}

다른 문서의 특정 문단으로 이동하려면 다음과 같이 작성한다.

{{{
[[문서명#문단명]]
[[문서명#문단명|표시할 글자]]
}}}

=== 존재하지 않는 문서 ===

아직 존재하지 않는 문서도 링크할 수 있다.

{{{
[[작성되지 않은 문서]]
}}}

이러한 링크는 작성이 필요한 문서 목록에 표시될 수 있다.

[clearfix]

== 외부 링크 ==

외부 사이트 주소를 연결하려면 다음과 같이 작성한다.

{{{
[https://example.com]
}}}

표시할 글자를 지정하려면 `|`를 사용한다.

{{{
[https://example.com|외부 사이트]
}}}

이중 대괄호 형식도 사용할 수 있다.

{{{
[[https://example.com|외부 사이트]]
}}}

주소는 `https://` 또는 `http://`로 시작해야 한다.

외부 링크는 출처, 공식 홈페이지, 관련 자료를 연결할 때 사용한다.

광고, 악성 사이트, 개인정보 수집 사이트로 연결해서는 안 된다.

[clearfix]

== 각주 ==

본문의 추가 설명이나 출처를 표시할 때 각주를 사용한다.

=== 기본 각주 ===

{{{
본문 내용[* 각주 내용]
}}}

=== 이름이 있는 각주 ===

{{{
본문 내용[*주석이름 각주 내용]
}}}

같은 각주를 다시 참조할 때는 다음과 같이 작성할 수 있다.

{{{
[*주석이름]
}}}

=== 각주 목록 ===

문서에서 각주 목록을 표시할 위치에 다음 매크로를 입력한다.

{{{
[각주]
}}}

일반적으로 문서 마지막 부분에 배치한다.

산삼위키에서는 각주 번호에 마우스를 올리면 각주 내용이 미리 표시되며, 번호를 누르면 해당 각주 위치로 이동한다.

=== 출처 각주 작성 예시 ===

{{{
해당 게임은 2025년에 정식 출시되었다.[* 공식 홈페이지의 출시 공지 참고]
}}}

가능하면 단순히 `인터넷 참고`라고 쓰기보다 자료명, 작성자, 날짜, 링크 등을 구체적으로 적는다.

[clearfix]

== 코드 ==

=== 짧은 코드 ===

짧은 명령어나 파일 이름은 중괄호 세 개로 감싼다.

{{{
{{{npm run dev}}}
}}}

=== 여러 줄 코드 ===

{{{
{{{
const message = "산삼위키";
console.log(message);
}}}
}}}

=== 문법 강조 ===

프로그래밍 언어를 지정하면 해당 언어에 맞는 색상으로 표시된다.

{{{
{{{#!syntax javascript
const wiki = "산삼위키";
console.log(wiki);
}}}
}}}

Python 예시:

{{{
{{{#!syntax python
wiki = "산삼위키"
print(wiki)
}}}
}}}

HTML 예시:

{{{
{{{#!syntax html
<h1>산삼위키</h1>
<p>위키 문서입니다.</p>
}}}
}}}

CSS 예시:

{{{
{{{#!syntax css
.wiki-title {
  font-weight: bold;
}
}}}
}}}

산삼위키는 highlight.js를 이용한 여러 언어의 문법 강조를 지원한다.

코드 블록 안에 비밀번호, 토큰, 개인정보를 넣지 않는다.

[clearfix]

== 수식 ==

TeX 수식은 `[math()]`를 사용한다.

{{{
[math(x^2+y^2=z^2)]
}}}

분수:

{{{
[math(\frac{a}{b})]
}}}

제곱근:

{{{
[math(\sqrt{x})]
}}}

합 기호:

{{{
[math(\sum_{i=1}^{n} i)]
}}}

행렬:

{{{
[math(\begin{pmatrix}1&2\\3&4\end{pmatrix})]
}}}

수식은 KaTeX로 렌더링된다.

일반 문장으로 충분히 설명할 수 있는 내용에 수식을 과도하게 사용하지 않는다.

[clearfix]

== 루비 문자 ==

한자나 외국어 위에 발음을 표시하려면 `[ruby()]`를 사용한다.

{{{
[ruby(本文, ruby=본문)]
}}}

예시:

{{{
[ruby(山蔘, ruby=산삼)]
}}}

한자 독음, 일본어 후리가나, 특수한 발음을 나타낼 때 사용할 수 있다.

[clearfix]

== 매크로 ==

=== 현재 날짜 ===

{{{
[date]
}}}

문서를 보는 시점의 현재 날짜를 표시한다.

=== 디데이 ===

{{{
[dday(2026-12-31)]
}}}

날짜 형식은 일반적으로 `YYYY-MM-DD`를 사용한다.

예시:

{{{
서버 개장일까지 [dday(2026-12-31)]
}}}

=== 페이지 수 ===

{{{
[pagecount]
}}}

위키에 저장된 페이지 수를 표시한다.

=== 플로트 정리 ===

{{{
[clearfix]
}}}

왼쪽이나 오른쪽에 배치된 이미지 또는 표 다음의 내용이 밀리는 현상을 정리할 때 사용한다.

[clearfix]

== 접기 ==

내용을 접었다 펼칠 수 있게 하려면 `#!folding` 블록을 사용한다.

{{{
{{{#!folding [ 펼치기 · 접기 ]
접힌 상태로 표시할 내용
}}}
}}}

제목은 자유롭게 바꿀 수 있다.

{{{
{{{#!folding [ 자세한 내용 보기 ]
여기에 자세한 설명을 작성한다.
}}}
}}}

접기 안에는 링크, 강조, 목록, 코드 등 다른 문법을 넣을 수 있다.

중요한 정보를 모두 접기 안에 숨기지 않는다.

문서의 핵심 내용은 펼친 상태에서도 확인할 수 있어야 한다.

[clearfix]

== 스타일 블록 ==

=== wiki style ===

특정 영역에 직접 스타일을 적용하려면 다음과 같이 작성한다.

{{{
{{{#!wiki style="text-align: center;"
가운데 정렬된 내용
}}}
}}}

배경색과 여백을 적용하는 예시:

{{{
{{{#!wiki style="background-color: #e8f5e9; padding: 10px;"
배경이 적용된 내용
}}}
}}}

스타일은 문서의 가독성을 높이기 위한 범위에서 사용한다.

지나치게 큰 글자, 움직이는 효과, 읽기 어려운 색상 조합은 피한다.

모바일 화면과 다크 모드에서도 읽을 수 있도록 작성하는 것이 좋다.

=== style 블록 ===

산삼위키 파서가 지원하는 범위에서 `#!style` 블록을 사용할 수 있다.

{{{
{{{#!style
스타일이 적용될 내용
}}}
}}}

실제로 적용되는 속성은 사이트의 보안 설정에 따라 제한될 수 있다.

[clearfix]

== HTML 블록 ==

HTML 코드를 직접 삽입하려면 `#!html` 블록을 사용한다.

{{{
{{{#!html
<div class="example">
  HTML 내용
</div>
}}}
}}}

일반적인 문서 작성에는 기본 위키 문법 사용을 권장한다.

다음과 같은 내용은 사용해서는 안 된다.

 * 외부 스크립트
 * 사용자 정보를 수집하는 코드
 * 자동 이동이나 자동 다운로드를 유도하는 코드
 * 악성 이벤트 속성
 * 위키 화면을 방해하는 요소
 * 다른 사용자를 속이는 가짜 로그인 화면

위험한 HTML 태그나 속성은 보안상 제거될 수 있다.

[clearfix]

== 조건문 블록 ==

조건에 따라 내용을 표시할 때 `#!if` 블록을 사용할 수 있다.

기본 구조:

{{{
{{{#!if 조건
조건을 만족할 때 표시할 내용
}}}
}}}

사용할 수 있는 조건식과 값은 산삼위키의 실제 구현에 따라 달라질 수 있다.

조건문은 일반 문서보다는 틀이나 특수 문서에서 사용하는 것을 권장한다.

[clearfix]

== 블록 중첩 ==

여러 줄 블록 안에 다른 블록을 넣을 수 있다.

예시:

{{{
{{{#!folding [ 코드 보기 ]
{{{#!syntax javascript
console.log("중첩된 코드");
}}}
}}}
}}}

각 블록은 반드시 대응되는 `}}}`로 닫아야 한다.

블록이 여러 단계로 겹칠 때는 들여쓰기를 맞추면 수정하기 쉽다.

[clearfix]

== 표 ==

=== 기본 표 ===

표의 각 셀은 `||`로 구분한다.

{{{
|| 첫 번째 칸 || 두 번째 칸 ||
|| 세 번째 칸 || 네 번째 칸 ||
}}}

각 줄이 하나의 행이 된다.

=== 표 작성 원칙 ===

 * 표는 정보를 비교하거나 요약할 때 사용한다.
 * 문장 전체를 표에 지나치게 많이 넣지 않는다.
 * 열이 너무 많으면 모바일 화면에서 읽기 어렵다.
 * 배경색과 글자색의 대비를 확인한다.
 * 표만으로 내용을 설명하기 어려우면 본문 설명을 함께 작성한다.

=== 표 안 줄바꿈 ===

{{{
|| 첫 번째 줄[br]두 번째 줄 || 일반 셀 ||
}}}

=== 표 정렬 ===

{{{
||<tablealign=left> 왼쪽 정렬 표 || 내용 ||
||<tablealign=center> 가운데 정렬 표 || 내용 ||
||<tablealign=right> 오른쪽 정렬 표 || 내용 ||
}}}

=== 표 너비 ===

{{{
||<tablewidth=100%> 제목 || 내용 ||
}}}

픽셀 단위도 사용할 수 있다.

{{{
||<tablewidth=600> 제목 || 내용 ||
}}}

모바일 화면을 고려하면 `%` 단위를 권장한다.

=== 표 테두리색 ===

{{{
||<tablebordercolor=#2e7d32> 내용 || 내용 ||
}}}

=== 표 배경색 ===

표 전체 배경색:

{{{
||<tablebgcolor=#e8f5e9> 내용 || 내용 ||
}}}

셀 하나의 배경색:

{{{
||<bgcolor=#2e7d32> 초록색 셀 || 일반 셀 ||
}}}

단축 형식:

{{{
||<#2e7d32> 초록색 셀 || 일반 셀 ||
}}}

=== 글자색 ===

{{{
||<bgcolor=#2e7d32><color=#ffffff> 흰색 글자 ||
}}}

=== 셀 너비 ===

{{{
||<width=30%> 왼쪽 셀 ||<width=70%> 오른쪽 셀 ||
}}}

=== 최대 너비 ===

{{{
||<maxwidth=500> 최대 너비가 제한된 셀 ||
}}}

=== 가로 셀 합치기 ===

{{{
||<-2> 두 칸을 합친 제목 ||
|| 왼쪽 || 오른쪽 ||
}}}

더 많은 칸을 합칠 수 있다.

{{{
<-3>
<-4>
}}}

=== 세로 셀 합치기 ===

아래쪽 방향 병합:

{{{
||<v|2> 두 행을 차지하는 셀 || 첫 번째 행 ||
|| 두 번째 행 ||
}}}

위쪽 기준 병합:

{{{
||<^|2> 두 행을 차지하는 셀 || 첫 번째 행 ||
|| 두 번째 행 ||
}}}

`N`에는 합칠 행의 수를 입력한다.

=== 정렬 속성 ===

구현된 표 속성에 따라 셀 안 내용을 정렬할 수 있다.

예시:

{{{
||<:> 가운데 정렬 ||
||<)> 오른쪽 정렬 ||
||<(> 왼쪽 정렬 ||
}}}

=== 표 안 블록 문법 ===

표 셀 안에 접기, 스타일, 코드 등의 블록을 넣을 수 있다.

{{{
|| 기능 || 설명 ||
|| 코드 예시 || {{{#!syntax javascript
console.log("산삼위키");
}}} ||
}}}

복잡한 블록을 넣을 때는 셀 구분자 `||`와 블록 종료 기호 `}}}`의 위치를 확인한다.

=== 정보상자 예시 ===

{{{
||<tablealign=right><tablewidth=380><tablebordercolor=#2e7d32><tablebgcolor=#ffffff><bgcolor=#2e7d32><color=#ffffff><-2> '''산삼위키''' ||
||<-2> [[파일:산삼위키 로고.png|width=100%]] ||
||<width=30%><bgcolor=#e8f5e9> 종류 || 게임 커뮤니티 위키 ||
||<bgcolor=#e8f5e9> 기반 || Vite ||
||<bgcolor=#e8f5e9> 배포 || Cloudflare Pages ||
||<bgcolor=#e8f5e9> 로그인 || Discord OAuth ||
}}}

=== 일반 정보표 예시 ===

{{{
||<tablealign=center><tablewidth=100%><tablebordercolor=#2e7d32><-2><bgcolor=#2e7d32><color=#ffffff> '''주요 기능''' ||
||<width=30%><bgcolor=#e8f5e9> 문서 편집 || 지원 ||
||<bgcolor=#e8f5e9> 문서 역사 || 지원 ||
||<bgcolor=#e8f5e9> 문서 ACL || 지원 ||
||<bgcolor=#e8f5e9> Discord 로그인 || 지원 ||
}}}

[clearfix]

== 파일과 이미지 ==

업로드된 파일은 `파일:파일명` 문서로 관리된다.

=== 이미지 삽입 ===

{{{
[[파일:산삼위키 로고.png]]
}}}

너비 지정:

{{{
[[파일:산삼위키 로고.png|width=300]]
}}}

화면 너비에 맞추기:

{{{
[[파일:산삼위키 로고.png|width=100%]]
}}}

=== 지원 파일 형식 ===

 * jpg
 * png
 * gif
 * webp
 * bmp
 * svg

파일 하나의 최대 크기는 4MB이다.

=== 파일 업로드 시 작성할 내용 ===

 * 파일 이름
 * 파일 설명
 * 출처
 * 제작자
 * 라이선스
 * 분류
 * 업로드 요약

파일 이름은 내용을 알아보기 쉽게 작성한다.

잘못된 예시:

{{{
image1.png
새 이미지.png
캡처123.png
}}}

올바른 예시:

{{{
산삼위키 로고 2026.png
게임명 공식 키아트.jpg
캐릭터명 프로필 이미지.webp
}}}

=== 파일 라이선스 ===

파일을 업로드하기 전에 사용할 수 있는 이미지인지 확인한다.

출처를 모르는 이미지는 업로드하지 않는 것이 좋다.

직접 제작한 파일은 직접 제작했음을 명시한다.

공식 게임 이미지나 홍보 자료는 해당 권리자의 정책에 따라 사용해야 한다.

=== 이미지 사용 원칙 ===

 * 문서 내용과 관련된 이미지만 사용한다.
 * 같은 이미지를 불필요하게 반복하지 않는다.
 * 지나치게 큰 이미지는 너비를 조절한다.
 * 혐오감을 주거나 개인정보가 포함된 이미지는 올리지 않는다.
 * 다른 이용자의 프로필 사진을 동의 없이 올리지 않는다.
 * 이미지로만 중요한 정보를 전달하지 않는다.

[clearfix]

== 영상 ==

산삼위키 편집기는 다음 영상 서비스의 URL을 자동으로 인식한다.

 * YouTube
 * Vimeo
 * 네이버 TV
 * 니코니코 동화

영상은 편집기 상단의 영상 삽입 도구를 이용하는 것을 권장한다.

영상을 삽입할 때는 영상의 내용을 짧게 설명한다.

삭제되거나 비공개로 전환될 가능성이 있으므로 문서 핵심 내용을 영상 하나에만 의존하지 않는다.

[clearfix]

== 틀과 include ==

다른 틀이나 문서 내용을 현재 문서에 포함하려면 `include` 문법을 사용한다.

{{{
[include(틀:틀이름)]
}}}

매개변수 전달:

{{{
[include(틀:틀이름,변수=값)]
}}}

여러 매개변수:

{{{
[include(틀:틀이름,이름=산삼위키,종류=게임 위키)]
}}}

틀 안에서 사용하는 정확한 변수명은 해당 틀 문서의 설명을 따른다.

틀을 수정하면 여러 문서에 동시에 영향을 줄 수 있으므로 신중하게 편집한다.

=== 국기 틀 ===

산삼위키는 여러 국가의 국기를 표시하는 `틀:깃발`과 `틀:국기`를 지원한다.

일반적인 호출 예시:

{{{
[include(틀:국기,국가명)]
}}}

{{{
[include(틀:깃발,국가명)]
}}}

실제 국가명과 매개변수 형식은 해당 틀의 설명 문서를 확인한다.

[clearfix]

== 분류 ==

문서를 주제별로 묶기 위해 분류를 사용한다.

분류 문법의 실제 입력 방식은 산삼위키 편집기와 구현된 파서 형식을 따른다.

문서를 분류할 때는 다음 원칙을 따른다.

 * 가장 직접적인 분류를 우선한다.
 * 같은 의미의 분류를 여러 개 만들지 않는다.
 * 너무 넓은 분류와 너무 좁은 분류를 동시에 남발하지 않는다.
 * 인물, 게임, 길드, 사건, 시스템 등을 적절히 구분한다.
 * 분류가 없는 문서는 미분류 문서 목록에 표시될 수 있다.

새 분류를 만들기 전에 기존 분류가 있는지 검색한다.

[clearfix]

== 문서 서술 방식 ==

=== 개요 문단 ===

문서의 첫 번째 본문 문단에는 대상을 간단히 설명한다.

예시:

{{{
'''산삼위키'''는 게임 커뮤니티를 위한 자체 위키 사이트이다.
}}}

첫 문장에는 문서 제목을 굵게 표시하는 방식을 권장한다.

=== 날짜 표기 ===

정확한 날짜를 알 수 있으면 다음과 같이 작성한다.

{{{
2026년 6월 14일
}}}

연도만 확인된 경우:

{{{
2026년
}}}

정확하지 않은 날짜를 임의로 만들지 않는다.

=== 숫자와 단위 ===

숫자와 단위는 문서 전체에서 통일한다.

예시:

{{{
4MB
300px
60%
2026년 6월
}}}

게임 안에서 사용하는 공식 단위가 있으면 해당 단위를 우선한다.

=== 인물과 이용자 서술 ===

실명, 학교, 주소, 전화번호, 계정 정보 등 개인정보를 작성하지 않는다.

공개적으로 활동하는 닉네임이라도 비방, 신상 추적, 사생활 폭로 목적으로 문서를 작성해서는 안 된다.

인물 문서는 해당 인물이 위키의 등재 기준을 충족하고 공개적으로 확인 가능한 정보가 충분할 때 작성한다.

=== 사건 서술 ===

사건 문서는 시간 순서와 사실관계를 구분하여 작성한다.

권장 구성:

{{{
== 개요 ==
== 배경 ==
== 진행 ==
== 결과 ==
== 반응 ==
== 이후 ==
}}}

확정된 사실과 주장, 추측을 구분한다.

{{{
운영진은 서버 오류가 원인이라고 밝혔다.
}}}

{{{
일부 이용자는 고의적인 조치였다고 주장했으나 확인되지는 않았다.
}}}

=== 게임 문서 권장 구성 ===

{{{
== 개요 ==
== 시스템 요구 사항 ==
== 게임 방식 ==
== 세계관 ==
== 등장인물 ==
== 콘텐츠 ==
== 평가 ==
== 업데이트 내역 ==
== 사건 사고 ==
== 관련 문서 ==
}}}

모든 문서가 이 구성을 반드시 따라야 하는 것은 아니다.

=== 길드 문서 권장 구성 ===

{{{
== 개요 ==
== 역사 ==
== 운영 방식 ==
== 구성원 ==
== 활동 ==
== 사건 사고 ==
== 관련 문서 ==
}}}

구성원의 개인정보나 비공개 대화 내용을 무단으로 작성하지 않는다.

[clearfix]

== 편집 요약 ==

문서를 저장할 때 편집 내용을 간단히 설명한다.

좋은 편집 요약 예시:

{{{
오탈자 수정
신규 문단 추가
공식 공지 기준으로 출시일 수정
표 디자인 정리
출처 각주 추가
문서 구조 개편
악성 편집 되돌림
}}}

좋지 않은 편집 요약 예시:

{{{
수정
ㅇㅇ
.
몰라
그냥
}}}

내용을 크게 변경하거나 삭제했다면 이유를 구체적으로 적는다.

편집 요약에는 욕설, 비난, 개인정보를 작성하지 않는다.

[clearfix]

== 미리보기와 비교 ==

=== 미리보기 ===

저장하기 전에 미리보기를 이용하여 다음 항목을 확인한다.

 * 문법이 제대로 닫혔는지
 * 표가 깨지지 않았는지
 * 링크가 올바른 문서로 연결되는지
 * 이미지 크기가 적절한지
 * 문단 단계가 올바른지
 * 오탈자가 없는지

=== 비교 ===

비교 기능을 이용하면 이전 내용과 현재 내용을 비교할 수 있다.

 * 추가된 내용
 * 삭제된 내용
 * 변경된 문장
 * 이동된 문단

대규모 편집을 하기 전과 저장하기 전에 비교 기능을 확인하는 것을 권장한다.

[clearfix]

== 문서 역사와 되돌리기 ==

문서가 수정될 때마다 리비전이 생성된다.

문서 역사에서는 다음 내용을 확인할 수 있다.

 * 편집 시각
 * 편집자
 * 편집 요약
 * 이전 판과의 차이
 * 리비전 내용

악성 편집, 실수, 문서 훼손이 발생한 경우 이전 리비전으로 되돌릴 수 있다.

되돌리기 권한은 사용자 역할과 ACL에 따라 제한될 수 있다.

되돌릴 때는 편집 요약에 이유를 적는다.

{{{
악성 편집 이전 판으로 되돌림
}}}

{{{
잘못 삭제된 문단 복구
}}}

반복적인 되돌리기 싸움은 금지한다.

[clearfix]

== 문서 이동 ==

문서 제목이 잘못되었거나 공식 명칭으로 바꿔야 하는 경우 문서 이동 기능을 사용한다.

내용을 복사하여 새 문서를 만들고 기존 문서를 비우는 방식은 문서 역사를 분리시키므로 피한다.

문서 이동 전 확인할 내용:

 * 이동할 새 제목이 정확한지
 * 같은 제목의 문서가 이미 존재하는지
 * 기존 링크가 어떻게 처리되는지
 * 넘겨주기가 필요한지
 * 관련 틀과 분류를 수정해야 하는지

문서 이동 권한은 ACL과 사용자 역할에 따라 제한될 수 있다.

[clearfix]

== 문서 삭제 ==

문서 삭제는 다음과 같은 경우에 검토할 수 있다.

 * 명백한 광고 문서
 * 의미 없는 시험 문서
 * 중복 문서
 * 개인정보 침해 문서
 * 악성 비방 문서
 * 저작권 침해 문서
 * 존재할 필요가 없는 잘못된 넘겨주기
 * 운영 정책을 위반한 문서

내용이 부족하다는 이유만으로 바로 삭제하기보다 보완하거나 다른 문서에 합칠 수 있는지 먼저 검토한다.

삭제 권한은 중재자 또는 관리자 등 역할에 따라 제한될 수 있다.

[clearfix]

== ACL ==

ACL은 문서를 읽거나 수정할 수 있는 사용자를 정하는 권한 규칙이다.

산삼위키에서 지원하는 주요 액션:

 * 읽기
 * 편집
 * 이동
 * 삭제
 * ACL 설정

대상 예시:

 * 모든 사용자
 * 로그인된 사용자
 * 정회원
 * 중재자
 * 관리자
 * 특정 사용자

ACL 규칙은 위에서 아래로 확인하며 처음 일치한 규칙이 적용된다.

관리자가 규칙 순서를 변경하거나 추가·삭제할 수 있다.

읽기 권한이 거부된 문서는 본문뿐 아니라 역사와 편집 화면도 차단될 수 있다.

=== ACL 사용 원칙 ===

 * 일반 문서는 지나치게 제한하지 않는다.
 * 반복적으로 훼손되는 문서는 일시적으로 보호할 수 있다.
 * 관리자용 문서나 정책 문서는 높은 권한을 요구할 수 있다.
 * 특정 이용자를 괴롭히기 위해 ACL을 사용해서는 안 된다.
 * ACL 변경 이유는 편집 요약이나 감사 기록에 남기는 것이 좋다.

[clearfix]

== 사용자 권한 ==

Discord 역할과 산삼위키 권한은 다음과 같이 연결될 수 있다.

||<tablealign=center><tablewidth=100%><tablebordercolor=#2e7d32><bgcolor=#2e7d32><color=#ffffff> 상태 ||<bgcolor=#2e7d32><color=#ffffff> 역할 ||<bgcolor=#2e7d32><color=#ffffff> 주요 권한 ||
|| 비로그인 또는 서버 미가입 || guest || 문서 읽기 ||
|| 서버 가입 || user || 일반 문서 편집 ||
|| 정회원 또는 신뢰 역할 || trusted || 보호 문서 편집, 이동 ||
|| 중재자 역할 || moderator || 삭제, 롤백 ||
|| 관리자 역할 || admin || ACL 변경, 사용자 차단 ||
|| 활성 차단 사용자 || guest로 강제 || 편집 제한 ||

실제 권한은 문서별 ACL과 전역 설정에 따라 달라질 수 있다.

[clearfix]

== 파일 업로드 지침 ==

파일을 올리기 전 다음 내용을 확인한다.

 1. 문서에 필요한 파일인지 확인한다.
 2. 같은 파일이 이미 올라와 있는지 검색한다.
 3. 파일 이름을 알아보기 쉽게 정한다.
 4. 출처와 제작자를 확인한다.
 5. 올바른 라이선스를 선택한다.
 6. 설명과 분류를 작성한다.
 7. 개인정보가 포함되지 않았는지 확인한다.
 8. 4MB 이하인지 확인한다.
 9. 지원 형식인지 확인한다.
 10. 업로드 뒤 생성된 `파일:파일명` 문서를 확인한다.

SVG 파일은 코드 형태의 이미지이므로 출처가 불분명하거나 위험한 SVG는 업로드하지 않는다.

[clearfix]

== 문서 훼손 ==

다음 행위는 문서 훼손으로 간주될 수 있다.

 * 문서 내용을 이유 없이 전부 삭제
 * 욕설이나 도배 추가
 * 관련 없는 광고 삽입
 * 잘못된 정보의 반복 추가
 * 링크를 악성 사이트로 변경
 * 표나 문법을 고의로 깨뜨림
 * 다른 이용자의 편집을 반복적으로 되돌림
 * 제목을 장난성 이름으로 변경
 * 파일을 혐오 이미지로 교체
 * 넘겨주기를 고의로 엉뚱한 문서로 연결
 * 출처를 조작하거나 삭제
 * 운영진을 사칭하는 내용 추가

문서 훼손을 발견하면 가능한 경우 이전 정상 리비전으로 되돌리고 편집 요약에 이유를 남긴다.

[clearfix]

== 금지되는 내용 ==

다음 내용은 작성하거나 업로드해서는 안 된다.

=== 개인정보 ===

 * 실명
 * 주소
 * 학교나 학급
 * 전화번호
 * 이메일
 * IP 주소
 * 계정 비밀번호
 * 로그인 토큰
 * 비공개 계정 정보
 * 동의 없이 공개한 얼굴 사진
 * 신상 추적 정보

=== 불법 또는 위험한 내용 ===

 * 불법 행위를 직접 실행하기 위한 구체적 안내
 * 악성코드 또는 계정 탈취 코드
 * 위키나 Discord 서버 공격 방법
 * 다른 이용자의 계정 침해 방법
 * 불법 복제물 배포 링크
 * 사기나 피싱 사이트
 * 위험한 파일 다운로드

=== 비방과 괴롭힘 ===

 * 특정 이용자에 대한 욕설
 * 집단 따돌림 유도
 * 허위 사실 유포
 * 외모나 신체에 대한 조롱
 * 개인정보 폭로
 * 특정 이용자를 공격하도록 선동
 * 증거 없는 범죄자 지목

=== 선정적·혐오스러운 내용 ===

게임 문서의 설명에 필요한 범위를 넘어선 선정적 이미지나 지나치게 잔혹한 자료를 올리지 않는다.

필요한 경우 경고 문구와 접기 기능을 사용하더라도 운영 정책을 우선한다.

[clearfix]

== 토론과 분쟁 해결 ==

편집 의견이 다를 때는 다음 순서를 권장한다.

 1. 문서 역사와 출처를 확인한다.
 2. 상대 편집자의 편집 요약을 확인한다.
 3. 객관적인 자료를 제시한다.
 4. 문서 토론이나 지정된 Discord 채널에서 의견을 나눈다.
 5. 합의가 어렵다면 중재자 또는 관리자에게 판단을 요청한다.
 6. 결정이 나오기 전까지 반복적인 되돌리기를 피한다.

토론에서는 사람보다 내용을 중심으로 말한다.

잘못된 예시:

{{{
당신은 아무것도 모르니까 편집하지 마세요.
}}}

올바른 예시:

{{{
공식 공지에는 출시일이 6월 12일로 적혀 있으므로 해당 날짜로 수정하는 것이 적절해 보입니다.
}}}

[clearfix]

== 새 문서 만들기 ==

새 문서를 만들기 전 다음 항목을 확인한다.

 1. 같은 내용의 문서가 이미 있는지 검색한다.
 2. 공식 명칭과 문서 제목을 확인한다.
 3. 최소한의 설명 자료가 있는지 확인한다.
 4. 출처를 준비한다.
 5. 적절한 분류와 관련 문서를 확인한다.
 6. 첫 문장에 대상을 명확히 설명한다.
 7. 문단 구조를 구성한다.
 8. 저장 전 미리보기를 확인한다.

=== 새 문서 기본 틀 ===

{{{
[목차]

== 개요 ==

'''문서명'''은 설명할 대상이다.

== 상세 ==

내용

== 특징 ==

내용

== 관련 문서 ==

 * [[관련 문서]]

[각주]
}}}

문서 종류에 맞게 필요 없는 문단은 제거하고 필요한 문단을 추가한다.

[clearfix]

== 문법 오류 점검 ==

저장하기 전 다음 항목을 확인한다.

=== 강조 문법 닫기 ===

잘못된 예시:

{{{
'''닫히지 않은 굵은 글자
}}}

올바른 예시:

{{{
'''닫힌 굵은 글자'''
}}}

=== 블록 닫기 ===

잘못된 예시:

{{{
{{{#!folding [ 열기 ]
내용
}}}

올바른 예시:

{{{
{{{#!folding [ 열기 ]
내용
}}}
}}}

=== 표 구분자 ===

잘못된 예시:

{{{
|| 셀 1 | 셀 2 ||
}}}

올바른 예시:

{{{
|| 셀 1 || 셀 2 ||
}}}

=== 링크 문법 ===

잘못된 예시:

{{{
[문서명]
}}}

올바른 예시:

{{{
[[문서명]]
}}}

=== 색상 대비 ===

읽기 어려운 예시:

{{{
||<bgcolor=#eeeeee><color=#ffffff> 밝은 배경의 흰색 글자 ||
}}}

읽기 쉬운 예시:

{{{
||<bgcolor=#2e7d32><color=#ffffff> 진한 배경의 흰색 글자 ||
}}}

[clearfix]

== 편집 전 최종 확인 ==

문서를 저장하기 전에 다음 항목을 확인한다.

 * 문서 제목이 정확한가?
 * 같은 문서가 이미 존재하지 않는가?
 * 첫 문장에서 대상을 설명했는가?
 * 문단 구조가 자연스러운가?
 * 사실과 의견을 구분했는가?
 * 확인되지 않은 내용을 단정하지 않았는가?
 * 출처가 필요한 내용에 각주를 달았는가?
 * 내부 링크와 외부 링크가 올바른가?
 * 표와 접기 블록이 정상적으로 닫혔는가?
 * 이미지의 출처와 라이선스를 확인했는가?
 * 개인정보가 포함되지 않았는가?
 * 편집 요약을 작성했는가?
 * 미리보기와 비교 화면을 확인했는가?

[clearfix]
